# Phase 1 Compliance Checklist - Mobile PWA Shell

Use this checklist before any store build work.

## A. Product and Legal Basics
- [ ] Privacy policy URL exists and is public.
- [ ] Terms of use URL exists and is public.
- [ ] Support/contact URL exists and is public.
- [ ] Company/legal publisher name is finalized.
- [ ] App display name finalized.

## B. Data and Permissions Disclosure
- [ ] Inventory of collected user data completed.
- [ ] Inventory of shared data with third parties completed.
- [ ] Retention policy documented.
- [ ] Permissions map completed (camera, storage, notifications, etc.).
- [ ] Purpose for each permission documented in user-facing language.

## C. Android Play Console Readiness
- [ ] Play Console app created.
- [ ] Data safety draft completed.
- [ ] Content rating questionnaire prepared.
- [ ] App category and tags selected.
- [ ] Test track strategy defined (internal/closed/open).

## D. iOS App Store Connect Readiness
- [ ] App Store Connect app created.
- [ ] Privacy nutrition labels draft completed.
- [ ] Review notes draft completed.
- [ ] TestFlight groups prepared.
- [ ] Demo/test account credentials prepared for reviewer.

## E. Security Baseline
- [ ] HTTPS-only endpoint confirmed.
- [ ] Domain allowlist decided for webview navigation.
- [ ] Session cookie settings verified for webview compatibility.
- [ ] Crash/analytics tools reviewed for sensitive-data redaction.
- [ ] Incident contact and rollback owner assigned.

## F. Native Value-Add Selection
Select at least one:
- [ ] Push notifications
- [ ] Native camera/file bridge improvements
- [ ] Biometric app lock
- [ ] Other approved native enhancement: __________________

## G. Go/No-Go
- [ ] Product owner sign-off
- [ ] Tech lead sign-off
- [ ] QA lead sign-off
- [ ] Compliance/legal sign-off

Result:
- [ ] GO to Phase 2
- [ ] NO-GO (list blockers): __________________
