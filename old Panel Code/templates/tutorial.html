Daily workflow

Dashboard aapka control center hai. Status count dekh kar decide karo team ko kis kaam par focus karna hai.

  • Pending records pehle clear karo, backlog mat badhne do.
  • Verified list ko hold mat karo, jaldi review complete karo.
  • Notifications ko action item ki tarah treat karo.

Groups aur tables strategy

  1. Group ko session ya cycle ke hisab se banao.
  2. Table naming pattern fix rakho (example: SchoolCode-Class-Section-Session-BatchNo).
  3. Production start hone ke baad table rename avoid karo.
  4. Har table ka owner fix karo.

Daily Workflow

Dashboard is your control center. Use status counts to decide team priority.

  • Clear Pending first so backlog does not grow.
  • Do not keep records long in Verified; review quickly.
  • Treat notifications as action items.

Groups and Tables Strategy

  1. Create groups by session or working cycle.
  2. Use one naming format for tables (example: SchoolCode-Class-Section-Session-BatchNo).
  3. Avoid renaming tables after production starts.
  4. Assign one owner per table.

Status flow aur approval check

Status Uddeshya Zimmedar
Pending Data correction aur cleanup Data Operator
Verified Review ho gaya, sign-off ka wait Reviewer
Approved Final output ke liye ready Client Admin
Downloaded Delivery tracking stage Client Admin ya Operator
Jaldi me Pending se direct Approved mat karo. Isse reprint load badhta hai.

Approve karne se pehle checklist

  • Name, class aur section source data se match kare.
  • Photo clear, centered aur sahi student ka ho.
  • ID number me digit missing ya extra na ho.
  • Mandatory fields blank na ho.
  • Same table me duplicate card na ho.

Tip: har table me reviewer sign-off ke baad hi final approval do.

Checklist Before Approve

  • Name, class, and section match source data.
  • Photo is clear, centered, and correct.
  • ID number has no missing or extra digits.
  • Mandatory fields are filled.
  • No duplicate card in same table.

Export, delivery aur reprint handling

Export aur delivery protocol

  1. Delivery stage ke hisab se Approved ya Download filter lagao.
  2. Required format me export karo, same button baar-baar mat click karo.
  3. Large job me background task complete hone do.
  4. Printer/school ko bhejne se pehle sample check karo.
  5. Export files ko standard naam se archive karo.

Reprint aur exception handling

  • Reprint sirf genuine correction case me karo.
  • Har reprint ka reason note karo.
  • Reprint ke baad final delivery se pehle card dobara verify karo.
  • Repeated reprints track karo, root cause nikaalo.
Suggested file name: Client-Table-Date-Format-Operator.
{% else %}

Exports, Delivery, and Reprint Handling

Export and Delivery Protocol

  1. Use Approved or Download filter based on delivery stage.
  2. Export in required format; do not click repeatedly.
  3. For large jobs, wait for background task completion.
  4. Check one sample before sending to printer/school.
  5. Archive files with standard naming.

Reprint and Exception Handling

  • Use reprint only for genuine correction cases.
  • Record reason for every reprint.
  • After reprint, verify card again before delivery.
  • Track repeated reprints to find root issue.
Suggested filename for exported bundles: Client-Table-Date-Format-Operator.
