## Deployment Checklist

Before uploading to Play Store:
1. **Code Review**: Verify no debug/test code in production build
2. **Version Bumps**: Update `VERSION.txt` (backend) and `app.json` (mobile)
3. **API Compatibility**: Ensure backend is running v3.19.0+
4. **Testing**: Run full mobile smoke tests on AAB
5. **Keystore**: Confirm `release.keystore` and `keystore.properties` are correct
6. **Artifact**: Generate AAB via `./gradlew bundleRelease` (signed automatically)
7. **Upload**: Use Play Console to upload AAB to internal/closed testing track
8. **Monitor**: Check crash-free sessions and ANR rates for 48 hours minimum

## ⚠️ CRITICAL — NEVER DELETE

**DO NOT DELETE** the `android_app/` directory. This contains:
- All mobile app source code (React Native / Expo)
- Android build configuration and signing setup
- Release keystore and properties
- All app assets and native modules

Deleting this directory will:
- Break all mobile app builds
- Lose signing credentials
- Make it impossible to upload to Play Store
- Require full recovery from git history

If accidentally deleted, restore via: `git checkout HEAD -- android_app/`
