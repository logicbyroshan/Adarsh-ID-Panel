# Generate Card Modal - Step 2 (Map) Deep Explanation

This document explains how Step 2 of the Generate Card modal works in the current codebase.

Scope: Step 2 only (the mapping/editor step), including the 4 left-side tools.

## 1) Where Step 2 Is Implemented

Primary files:

- `templates/cardprint/generate-card.html`
- `static/js/cardprint/generate-card.js`
- `cardprint/views.py`
- `cardprint/urls.py`

Step 2 UI is rendered entirely from JavaScript via `renderStep2()` in `static/js/cardprint/generate-card.js`.

## 2) Step 2 Position In The Flow

The modal has 3 steps:

1. Step 1: Setup
2. Step 2: Map
3. Step 3: Generate

Navigation behavior:

- `handleStepNext()` moves to next step.
- When entering step 2, `ensureStep2DraftInitialized()` is called.
- Step 1 currently has no hard validation (`step1Valid()` returns true), so users can always enter Step 2.

## 3) Step 2 Layout Overview

`renderStep2()` builds three main columns:

1. Left: compact tool rail (4 drawing tools + panel launch icons)
2. Center: canvas workspace + rulers + guides + minimal top bar
3. Right: modular floating panel host (Text / Align / Layers)

Important top controls in Step 2:

- Save template
- Zoom in/out/fit
- Undo / Redo
- Front/Back side switch (for two-sided templates)

Controls moved to the Text panel:

- Unit selector (`mm`, `cm`, `in`)
- Snap grid amount in mm
- Lock/Unlock guides
- Merge preview toggle
- Insert merge field
- Auto-map fields + report
- Insert photo field

## 4) Step 2 Draft Data Model

All editing is done on an in-memory draft object:

- `state.templateDraft.canvas`
  - `width`, `height`
  - `guides[]`
- `state.templateDraft.elements[]`

Each element is normalized by `normalizeDraftElement(...)` to keep safe defaults and bounds.

Supported element types:

- `text`
- `image`
- `rectangle`

Each element has a stable internal id (`__id`) generated by `nextDraftElementId()`.

Selection model:

- Primary selection: `state.draftSelectedElementId`
- Multi-selection set: `state.draftSelectedElementIds`
- Guide selection: `state.draftSelectedGuideId`

Text model details (important for current artistic text behavior):

- Text elements store both `text` and `label` for compatibility; visible text is resolved through helper logic.
- Text mode is normalized via `textType` (`artistic` or `paragraph`) with backward support for legacy `textMode`.
- Artistic text stores transform properties separately from base glyph box:
  - Base box: `width`, `height`
  - Transform: `scaleX`, `scaleY`, `rotation`, `skewX`, `skewY`
- Artistic selection/render/alignment now use transformed bounds (AABB from matrix math), not plain `width/height` only.

## 5) The 4 Step 2 Tools (Main Requirement)

The tool buttons are rendered in `renderStep2()` with `data-action="set-draft-tool"`:

1. `select`
2. `text`
3. `photo`
4. `rectangle`

### 5.1 Select Tool

Purpose:

- Select elements
- Multi-select elements
- Move selected elements
- Resize, rotate, and skew via mode-aware handles
- Marquee box selection
- Enter inline text edit on double-click (for text elements)

How it works:

- Click element: selects it.
- Shift+click: toggle add/remove selection membership.
- Drag element: moves element (or group if multi-selected).
- Drag handle:
  - Resize mode -> resize
  - Rotate mode -> corner handles rotate, side handles skew
- Click selected element again (without dragging): toggles `resize` <-> `rotate` transform mode.
- Drag on empty canvas: draws selection box.
- Marquee selection updates live while dragging (intersect-based).
- Shift while moving: axis-locks movement to X or Y.
- Double-click text element: enters inline editing (`setDraftInlineTextEditing(...)`).
- Double-click selected non-text element: toggles transform mode.
- In multi-selection, clicking an already-selected element sets it as the key object for key-object alignment mode.

Artistic text specifics inside Select tool:

- Artistic handle resize changes transform scale (`scaleX`/`scaleY`) rather than mutating the base text box directly.
- Font metrics are preserved during artistic resize, giving Corel-style transformed text behavior.
- Marquee selection and align/distribute operations use transformed artistic bounds.
- Inline editing for artistic text is rendered with the same transform (scale/rotate/skew), so edit view and normal view match.
- Transform drag feel is tuned for control:
  - Rotate drag uses a damped sensitivity curve with tiny-angle jitter suppression.
  - Skew drag uses reduced per-span sensitivity with shift snapping.
  - Handles use larger visual/hit targets for easier grabbing.
- Key object is visually highlighted with a distinct selection outline.

Supporting functions/events:

- `flowRoot.addEventListener('mousedown', ...)`
- `window.addEventListener('mousemove', ...)`
- `window.addEventListener('mouseup', ...)`
- `selectDraftElementsByBox(...)`
- `applyDraftResizeDrag(...)`

### 5.2 Text Tool

Purpose:

- Add text elements by click-drag or quick click.

Creation behavior on mouse up:

- If drag distance is large (`dx >= 14 || dy >= 14`):
  - Creates a paragraph text box.
- If drag distance is small:
  - Creates artistic text (single-line style).

After creating artistic text:

- Inline editing is automatically started, and for transformed artistic text the editor is shown with the same transform matrix.

Creation path:

- Mousedown on canvas stores `state.draftTextDrag`
- Mousemove updates drag box preview
- Mouseup commits with `addDraftElement('text', ...)`

### 5.3 Photo Tool

Purpose:

- Insert image placeholder/photo field elements.

Behavior:

- This tool inserts on **double-click on canvas**.
- It calls `addPhotoPlaceholderElement(...)`.
- Placeholder size is based on 19x25 mm proportional to current canvas real dimensions.
- Inserted position is centered around pointer.
- After inserting via canvas double-click, tool auto-switches back to `select`.

Also related:

- Top control `Insert Photo Field` can insert/match image field directly (`insertPhotoFieldElement(...)`).

### 5.4 Rectangle Tool

Purpose:

- Draw visual rectangle blocks/background shapes.

Behavior:

- Drag on canvas to draw rectangle.
- `Shift` during drag locks square aspect.
- On mouseup, if box is large enough, commits `addDraftElement('rectangle', ...)`.
- Double-click on empty canvas creates full-canvas rectangle.

Visual style:

- Colorized rectangle with translucent fill from color token.
- Uses rectangle-specific CSS class and render branch.

## 6) Real-Time Interaction Pipeline

For all tools, interaction uses the same low-level flow:

1. `mousedown` starts operation and stores drag state
2. `mousemove` updates temporary visual state
3. `mouseup` commits mutation and ends history transaction

Temporary drag states:

- `draftTextDrag`
- `draftRectDrag`
- `draftSelectDrag`
- `draftDragging` (element move)
- `draftResizeDragging`
- `draftGuideDragging`

Temporary overlay during drag is rendered by `renderDraftInsertGuideHtml()`.

## 7) Guides, Rulers, Snap, And Precision

Step 2 includes ruler-based guide creation:

- Drag from top ruler to create horizontal guide
- Drag from left ruler to create vertical guide
- Drag guide to move it
- Double-click guide to remove it

Guide lock:

- `toggle-guides-lock` disables guide interactions.

Snap:

- Snap amount is user-defined in mm (`state.draftSnapMm`).
- Position/drag operations use `snapCanvasValueToGrid(...)`.

Units:

- UI display unit can be `mm`, `cm`, or `in`.
- Internal math still uses canvas units.

## 8) Side-Aware Editing (Front/Back/Both)

For two-sided mode:

- Canvas is rendered as combined front + back area.
- Elements can target `front`, `back`, or `both`.
- Active editing side is tracked in `state.draftActiveSide`.

Single-sided mode forces effective side to `front`.

## 9) Merge Field Features Inside Step 2

Step 2 includes mapping helpers beyond manual tool placement:

- Insert merge field into text element
- Auto-map fields from element labels
- Open auto-map report modal
- Insert photo field
- Merge preview mode (`{{Field Name}}` style visualization)

Key functions:

- `syncSelectedElementField(...)`
- `autoMapDraftFields(...)`
- `insertPhotoFieldElement(...)`

## 9.1) Align & Distribute Reference Modes

Step 2 now supports Corel-style alignment reference behavior via properties controls:

- `Selection`: align relative to combined transformed selection bounds
- `Page`: align relative to canvas/page bounds
- `Key Object`: align relative to selected key object bounds

Key object behavior:

- Available when 2+ elements are selected
- Set by clicking one selected element again
- Key object does not move during key-object alignment/distribution operations
- During key-object distribution, non-key elements are distributed independently on each side of the key object (left/right or top/bottom), so the key object acts as a fixed anchor.

Distribution variants are selectable:

- `Spacing` (equal edge gaps)
- `Centers` (equal center-to-center spacing)
- `Edges` (equal edge-to-edge stepping)

Distribution remains transform-aware (uses rendered/transformed bounds).

## 9.2) Layers (Z-Index) & Modular Panels

Step 2 now includes a Corel-style layer stack model on each element:

- `zIndex` controls stacking order (higher value renders on top)
- `visible` controls render/hit-test inclusion
- `locked` prevents selection and transform operations

Render order:

- Elements are sorted by `zIndex` before canvas render.
- New elements are created at top stack (`max zIndex + 1`).
- Layer reorders normalize stack to sequential values.

Overlap click behavior:

- Hit selection now resolves to the highest `zIndex` selectable object at pointer location.

Layer operations (multi-select aware):

- Bring Forward
- Send Backward
- Bring To Front
- Send To Back

For multi-selection, relative ordering inside the selected group is preserved while moving in stack as a block.

Right-side modular panels:

- `Text` panel (existing object/text controls)
- `Align` panel
- `Layers` panel (object manager)

Panel behavior:

- Hidden by default
- Open from grouped left toolbar panel icons
- Single-active panel model (opening one closes others)
- Close from panel header close button
- Persisted between sessions (last active panel remembered)

Layers panel supports:

- Select layer row (for selectable items)
- Toggle visibility (eye)
- Toggle lock (lock)
- Drag row reorder (updates `zIndex`)
- Stack action buttons (front/back/forward/backward)

## 10) Undo/Redo And History Transactions

History is transaction-based and bounded (`maxDepth: 15`).

- `beginDraftHistoryTransaction()` at interaction start
- Mutation(s) happen
- `endDraftHistoryTransaction()` on commit/end

Notes:

- Continuous drags and inline typing are grouped transactionally to keep undo/redo practical.
- Break-text (`Ctrl/Cmd + K`) runs in a single transaction (split + replace + selection update).

Undo/Redo actions:

- Button actions `undo-draft`, `redo-draft`
- Keyboard:
  - `Ctrl/Cmd + Z` undo
  - `Ctrl/Cmd + Shift + Z` redo
  - `Ctrl/Cmd + Y` redo

## 11) Keyboard Shortcuts In Step 2

Main shortcuts:

- `Delete` / `Backspace`: remove selected element or guide
- Arrow keys: nudge selected element (Shift for larger step)
- `Ctrl/Cmd + D`: duplicate selected element
- `Ctrl/Cmd + S`: save template
- `Ctrl/Cmd + B`: toggle bold (text)
- `Ctrl/Cmd + I`: toggle italic (text)
- `Ctrl/Cmd + K`: break selected artistic text into chars/words (prompt-based)
- `Ctrl/Cmd + ]`: bring selected layer(s) forward
- `Ctrl/Cmd + [`: send selected layer(s) backward
- `Ctrl/Cmd + Shift + ]`: bring selected layer(s) to front
- `Ctrl/Cmd + Shift + [`: send selected layer(s) to back
- `Ctrl/Cmd + Alt + T`: toggle Text panel
- `Ctrl/Cmd + Alt + A`: toggle Align panel
- `Ctrl/Cmd + Alt + L`: toggle Layers panel
- `Ctrl/Cmd + +/-/0`: zoom controls
- `Escape`: cancel active operation, switch to select tool, and reset transform mode to `resize`
- Hold `Space`: pan mode on canvas stage

Break-text behavior (`Ctrl/Cmd + K`):

- Works only when the selected element is artistic text.
- Prompts for split mode (`chars` or `words`).
- Creates new artistic text elements per segment with copied style/transform properties.
- Places segments using transformed axis progression so rotated/skewed text stays visually consistent.

## 12) Saving Step 2 Changes

Save button behavior:

- Header button appears only in Step 2.
- Calls `triggerSaveDraftTemplate()` -> `saveDraftTemplate()`.

Save API path used by Step 2:

- Frontend currently saves through template endpoints:
  - `POST /api/templates/<table_id>/` (create)
  - `PUT /api/templates/<template_id>/` (new version save/update)

Server-side save behavior:

- Template JSON is sanitized and validated.
- A new template version is created on update flows.
- Returned payload includes updated template list and active template id.

## 13) Backend Endpoints Relevant To Step 2

Defined in `cardprint/urls.py` and implemented in `cardprint/views.py`:

- `GET /api/template/<template_id>/` -> load full template detail
- `POST /api/templates/<table_id>/` -> create template
- `PUT /api/templates/<template_id>/` -> save/update template version
- `POST /api/generate-card/table/<table_id>/template/upload-pdf/<side>/` -> upload design background PDF
- `POST /api/generate-card/table/<table_id>/template/clear-pdf/<side>/` -> clear side PDF

Note: Step 2 itself is editor/mapping. Final PDF generation happens in Step 3 (`/api/generate-card/table/<table_id>/generate/`).

## 14) Practical Mental Model

Think of Step 2 as a mini design editor with:

- A normalized JSON draft model
- Tool-driven creation and manipulation of elements
- Side-aware placement logic
- Mapping helpers for schema fields
- Transactional undo/redo
- Template version save pipeline

That is why the Step 2 behavior feels like a lightweight canvas/label designer instead of a plain form.
